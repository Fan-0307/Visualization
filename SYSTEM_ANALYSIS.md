## 二、视图重要性评估

### Tab1 模型对比 ⭐⭐⭐⭐⭐ — 不可移除
**角色**: 系统入口，概览视图。准确率总览 + 按问题类型拆解 + 柱状图。
**价值**: 评审者第一眼建立全局认知——"谁好谁差、哪种问题难"。
**去留**: 保留。可与Tab3合并。

### Tab2 Error Matrix & Attention ⭐⭐⭐⭐⭐ — 不可移除
**角色**: 导航中枢。199×4矩阵，颜色编码正确/错误，点击展开该样本的所有模型热力图。
**价值**: 唯一能同时看到"哪个样本难、哪个模型在哪个样本上错"的视图。点击展开是 details-on-demand 的核心实现。
**去留**: 保留。将Tab3的正误模式分析和Tab4的差异图能力整合进来。

### Tab3 正确/错误模式 ⭐⭐⭐ — 可合并
**角色**: 缩略图展示某模型正确/错误样本 + 按问题类型的堆叠柱状图。
**问题**: 柱状图与Tab1功能重叠（都是 question_type 维度）；缩略图在Tab2点击后也能看到。
**去留**: 合并到Tab1（柱状图）+ Tab2（缩略图面板）。

### Tab4 聚合注意力对比 ⭐⭐⭐⭐ — 核心诊断（依赖token数据）
**角色**: 多模型热力图并排对比 + 正确-错误差异图。
**价值**: "为什么A对B错"的空间解释——差异图红蓝编码直接揭示注意力分歧区域。
**问题**: 依赖 attn_index.json 数据线。如果砍掉token数据，差异图无法基于原始 attention 向量计算。
**替代**: Tab2底部面板可加 diff 模式，用 photos/ 的PNG热力图做像素级相减。

### Tab5 逐层Token注意力 ⭐⭐⭐ — 可牺牲
**角色**: 逐层逐token浏览注意力，token pill + sparkline + 播放动画。
**问题**: 浏览效率低（96样本×28层×N个token），展示性 > 分析性。技术实现复杂但分析产出有限。
**去留**: 可移除或降级为Tab4的展开面板。

### Tab6 逐层演化 V3 ⭐⭐⭐⭐⭐ — 核心差异化
**角色**: 注意力从浅层到深层、跨组件（vision_tower → qformer → language_model）的演化追踪。
**数据**: 独立数据线（layer_evo），4模型含CLIP，828样本。
**价值**: 架构层面的独特洞察——不同模型的组件切换点、注意力聚焦/扩散趋势。这是与其他工具（VL-InterpreT）最大的差异点。
**去留**: 保留。

### Tab7 Sink相关性 ⭐⭐⭐⭐ — 量化验证（缺 sink_score）
**角色**: 注意力Sink浓度与正确率的量化关联分析。
**价值**: 唯一从"定性观察"上升到"定量检验"的视图——有点二列相关、逐模型统计。
**问题**: 需要 sink_score 字段，attn_index.json 中该字段当前为空。可以改用 layer_evo 数据的 attention grid 自行计算 Sink Score。
**去留**: 保留但需重构数据源。

### 推荐最终布局

| Tab | 名称 | 数据源 | 角色 |
|-----|------|--------|------|
| 1 | 模型总览 | results.json | 概览：准确率计分卡 + 问题类型拆解 + 模型间一致性 |
| 2 | Error Matrix & 诊断 | results.json + photos/ | 导航中枢 + 热力图对比 + 差异图 + 一键诊断 |
| 3 | 逐层演化 | layer_evo | 架构分析 + 组件切换 + 双模型对比 |
| 4 | 注意力质量 | layer_evo (重算) | Sink/集中度/覆盖率 vs 正确率 |

4 Tab 闭环：**概览 → 定位 → 诊断 → 验证**

## 三、可挖掘的统计与可视化特征

### 3.1 从 results.json 可直接计算的

#### 1. 模型间一致性矩阵（4×4热力图）
```
          Qwen  LLaVA  Salesforce  CLIP
Qwen      1.00  0.85   0.72        0.58
LLaVA            1.00   0.74        0.61
Salesforce              1.00        0.67
CLIP                                 1.00
```
值 = 两个模型在所有样本上给出相同答案的比例。可拆分：只看都答对的样本 vs 都答错的样本 → 揭示模型在"容易题"上一致、在"难题"上分歧。

**可视化**: 对称热力图，下三角/上三角各显示一个统计量。

#### 2. 答案多样性指数（per sample）
每个样本 4 个模型给出几个不同答案？1个（全一致）→ 4个（各说各话）。
- 高多样性 + 全错 → 标注可能有问题的样本
- 高多样性 + 部分对 → 模型能力差异的典型样本

**可视化**: 在Tab2矩阵上加一列"分歧度"颜色条。

#### 3. 正确率 × 问题类型 × 模型的交互效应
不是简单的分组柱状图，而是 4×8 热力图矩阵：
```
         yes/no  what  what color  how many  where  who  which  other
Qwen     97%     80%   100%        74%       60%    100%  50%    57%
LLaVA    94%     75%   100%        58%       60%    100%  100%   71%
Salesf.  56%     60%   70%         44%       40%    50%   50%    57%
CLIP     0%      15%   10%         0%        40%    0%    0%     0%
```
一眼看出：CLIP 在 yes/no 上全军覆没（0%），但在 where 上有 40% → 可深挖。

**可视化**: 行列热力图 + 聚类（对行和列分别排序，相似模型/类型靠拢）。

#### 4. 置信度校准曲线
`results.json` 有 `confidence` 字段。绘制：横轴 = 置信度区间，纵轴 = 该区间内实际正确率。
- 理想: 对角线（conf=0.8 → 实际正确率≈80%）
- 过度自信: 曲线在对角线下方
- 可对比 4 个模型的校准曲线

#### 5. 错误类型推断
不依赖人工标注，用规则推断：
- 模型答案 = ground_truth 的反义词（yes↔no）→ "语义反转"
- 模型答案是另一个常见物体 → "对象混淆"
- 模型答案是数字偏差 → "计数错误"

**可视化**: 堆叠柱状图，每种错误类型一个颜色。

### 3.2 从 layer_evo 数据可计算的

#### 6. 注意力集中度演化曲线（已有Tab6基础，可增强）
当前 Tab6 已有个体样本的集中度/熵/Gini/Top5曲线。可增加：
- **模型平均曲线**：聚合该模型所有样本，画出"平均集中度×层深"的置信带
- **正确 vs 错误分离曲线**：同一模型，正确样本和错误样本的演化曲线是否不同？

这是 **系统级洞察** 而非单样本浏览。

#### 7. 组件边界"冲击"强度排行
`adjacent_layer_differences` 有 L1 和 KL。聚合所有样本：
- 哪个组件切换点（如 vision_tower → language_model）的 KL 散度最大？
- 不同模型的组件边界强度有何不同？（BLIP 的 qformer→decoder 切换可能最大）

**可视化**: 横向柱状图，按 KL 降序排列所有组件边界。

#### 8. 模型间注意力空间相关性
同一张图，模型 A 和模型 B 的 attention grid 有多大程度重叠？
- 先将不同分辨率的 grid 统一 resize 到相同尺寸
- 计算对应 patch 的 Pearson/Spearman 相关系数
- 聚合所有样本 → 得到 4×4 的"注意力模式相似度矩阵"

**发现可能的洞察**: "Qwen 和 LLaVA 的注意力模式高度相关（r=0.72），但 Salesforce BLIP 相对独立（r=0.45）——这可能反映了 decoder-only vs Q-Former 架构的差异。"

#### 9. 注意力空间分布偏差
对每个模型，聚合所有样本的注意力 grid 的平均值，得到一张"模型先验注意力地图"——不依赖于问题，模型天生更关注哪些图像区域？
- 中心偏置（center bias）
- 底部偏置（text bias — 如果模型习惯看字幕区域）
- 均匀分布（无偏置 → 更好的泛化）

**可视化**: 4个模型各一张聚合热力图，一眼看出模型的"视觉偏好"。

#### 10. 网格分辨率 vs 问题类型准确率
不同模型输出不同分辨率的视觉网格（Qwen: 46×34, BLIP: 24×24 等）。高分辨率模型在细粒度问题（what color, how many）上是否显著更好？

**可视化**: 散点图，横轴=视觉token数，纵轴=准确率，按问题类型着色。

#### 11. 注意力流动的"停滞"检测
如果连续 3 层以上的 attention grid 几乎不变（L1 距离 < 阈值），说明信息在这些层没有进一步加工——可能是冗余层或"过拟合层"。

**可视化**: 在演化曲线上标注"停滞区"（灰色底色）。

#### 12. Sink Score 重算（不依赖 attn_index.json）
用 layer_evo 数据的 `raw_visual_grid` 直接计算：
```
Sink Score = sum(top-1% patches) / sum(all patches)
```
对每个模型的每个样本，在最后 1/3 层上取平均。然后与 correctness 做点二列相关。
- 这个计算完全在 Python 预处理中完成，输出到 `layer_evo_index.json` 或独立 JSON。
- 与 Tab7 的 SinkCorrelationView 保持兼容。

### 3.3 从 photos/ 可提取的

#### 13. 注意力重心追踪
对每个热力图 PNG，计算像素强度的加权质心 (cx, cy)。对比正确/错误样本的重心分布：
- 正确样本的重心是否更集中在物体区域？
- 错误样本的重心是否偏向边缘/背景？

### 3.4 跨数据源的合成特征

#### 14. 样本难度评分
综合多个信号给每个样本打分：
- 平均正确率（越低越难）
- 模型答案一致性（越分歧越难）
- 注意力集中度（过高可能说明图像信息不足）

用这个评分可以在 Tab2 中排序样本，最难的排前面。

#### 15. 模型"视觉依赖度"
对比每个模型在"纯文本可答"的样本（yes/no 居多）vs "必须看图"的样本（what color, how many）上的准确率差异。
- 差距大 → 模型真正在用视觉信息
- 差距小 → 模型可能依赖语言先验，视觉输入没发挥作用

## 四、交互增强方案

### 4.1 跨视图联动（L1-L4 四层）

**L1 基础选中联动**（sample_id 驱动）
- Tab2 点击格子 → Tab4/Tab6 自动加载同一样本
- Tab4 切换样本 → Tab2 高亮对应列

**L2 模型级联动**（model 驱动）
- Tab1 点击模型 → Tab2 高亮该模型行、Tab7 滚动到该模型卡片
- Tab7 点击模型卡片 → Tab1 高亮该模型

**L3 分析线索联动**（question_type / layer 驱动）
- Tab1 点击问题类型 → Tab2 过滤到该类型样本
- Tab5 层滑块 → Tab6 sparkline 同步标记
- Tab6 组件交界点击 → Tab5 自动切层

**L4 对比模式联动**（双模型协同）
- Tab1 选择两个模型 → Tab4 差异图切换为 "模型A vs 模型B"
- Tab2 选中样本 → Tab6 双模型模式自动加载两侧

实现: 全局 `reactive` store + `provide/inject`，source 标记避免循环触发。

### 4.2 直接操纵

- **热力图区域圈选 → Token 溯源**: Tab4 热力图上拖拽框选区域 → 高亮最关注该区域的 token
- **层深度实时插值**: 拖拽层滑块时在相邻层间线性插值，热力图连续过渡
- **双模型对比滑杆**: 两张热力图叠加，拖拽分界线 reveal/hide（before/after slider）

### 4.3 智能辅助

- **一键诊断文本**: 点击错误样本 → 自动生成"为什么错"的描述（模板 + 数据查询）
- **注意力异常检测**: 标注注意力模式偏离模型均值 > 2σ 的样本（⚡标记）
- **探索路径面包屑**: 记录用户操作历史，可回退可分享

### 4.4 双 Tab 分屏

导航栏加"分屏"按钮 → 左右各一个独立 Tab 面板，可同时操作两个视图并保持联动。

## 七、可复现性说明

### 数据生成
- `run_all.sh` 一键运行4个模型，需 GPU（~30min/模型）
- `build_index.py` 构建索引
- 分层抽样 199 样本，8 种问题类型均衡

### 前端运行
```bash
npm install    # 或 pnpm install
npm run dev    # 开发服务器
npm run build  # 生产构建
```

### 数据分发
- `visualization_data.tar.gz`（359M）包含所有 gitignored 数据
- 解压到项目根目录即可运行
