## 当前系统状态

当前可视化系统是一个 Vue + Vite 单页应用，已有 6 个主要视图：

1. 模型对比：展示整体准确率、按问题类型准确率、柱状图。
2. Error Matrix & Attention：按样本和模型展示正确/错误矩阵，点击后查看热力图。
3. 正确 / 错误模式：按模型展示答对/答错样本缩略图，以及问题类型分布。
4. 聚合注意力对比：多模型聚合注意力热力图和正确/错误差异图。
5. 逐层 Token 注意力：按 token、层、模型查看注意力。
6. 逐层演化 V3：展示不同模型跨层、跨组件的注意力演化。

数据层目前主要有三条线：

- `src/data/processed_data/results.json`：199 个样本 × 4 个模型，包含准确率、预测结果、问题类型。
- `src/data/attn_index.json` + `public/data/attn/`：token 级注意力数据，主要支持聚合注意力和逐 token 注意力。
- `src/data/layer_evo_index.json` + `public/data/layer_evo/`：逐层演化数据，支持架构层分析。

注意：之前 `question_type` 已经修复，前端现在应读取 `results.json` 中的真实问题类型，而不是固定为 `All`。

当前问题：

- Error Matrix 信息量大，但样本多时不容易快速定位典型案例。
- 目前只有“只显示有错误的样本”，筛选能力不足。

建议修改：

- 增加错误类型筛选：全部样本、至少一个模型错误、全部模型错误、仅部分模型错误。
- 增加排序：默认排序、错误模型数量从高到低、模型预测分歧度从高到低。

实现方式：

- 基于 `props.data` 直接计算，不需要新数据。
- 错误模型数量 = 同一 `sample_id` 下 `correct === false` 的数量。
- 分歧度可以先用预测答案去重数量近似：
  - 4 个模型答案都不同，分歧度最高。
  - 所有模型答案相同，分歧度最低。

预计工作量：2 小时

## 推荐答辩叙事

建议用下面的顺序讲：

1. 全局表现：Qwen2-VL 和 LLaVA 整体领先，BLIP2 居中，CLIP 明显较弱。
2. 问题类型差异：不同模型在 `how many`、`what color`、`yes/no` 上表现不同。
3. 样本定位：通过 Error Matrix 找到全错样本和模型分歧样本。
4. 注意力解释：通过热力图和差异图说明正确模型和错误模型关注区域不同。
5. 架构分析：通过逐层演化展示不同模型组件切换时注意力如何变化。

## 3 分钟讲稿草案

```text
大家好，我们的项目是 VLM Attention Analysis，目标是分析视觉语言模型不仅答得对不对，还要看它们在回答问题时到底关注了图像的哪些区域。

我们选取了 199 个 VQA 样本，覆盖 8 类问题类型，并比较了 Qwen2-VL、LLaVA、BLIP2 和 CLIP 四个模型。系统包括三条数据线：模型预测结果、token 级注意力，以及逐层注意力演化。

首先在模型对比视图中，可以看到 Qwen2-VL 和 LLaVA 整体准确率最高，分别达到 84.4% 和 79.4%。进一步按问题类型拆解后，可以发现不同模型的能力差异并不均匀，例如 what color 和 how many 类型对视觉 grounding 的要求更高。

接下来在 Error Matrix 中，我们可以快速定位哪些样本被哪些模型答错。点击一个样本后，系统会展示原图、问题、标准答案、各模型预测以及对应注意力热力图。这样可以从单纯的准确率，进一步进入错误诊断。

最后，逐层演化视图展示注意力如何在模型不同层之间变化。它帮助我们观察视觉编码、跨模态融合和语言生成阶段的注意力迁移，是系统区别于普通结果统计的核心部分。

总结来说，我们的系统将模型表现、问题类型、样本级错误和层级注意力演化结合起来，用于支持多模型 VLM 的可解释性比较分析。
```
